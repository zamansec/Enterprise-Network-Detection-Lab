# Detection 004 - Suspicious HTTP User-Agent

## Objective

Identify HTTP requests using suspicious or uncommon User-Agent strings that may indicate malware, automated tools, or offensive security frameworks.

---

## Data Source

Zeek

Log:

- http.log

---

## SPL Query

```spl
index=zeek sourcetype=zeek:http
| search user_agent="*curl*" OR user_agent="*wget*" OR user_agent="*python*" OR user_agent="*PowerShell*" OR user_agent=""
| table _time id.orig_h id.resp_h host uri user_agent
| sort - _time
```

---

## Explanation

This query searches for HTTP requests generated by tools commonly used in automation, penetration testing, or malware.

Examples include:

- curl
- wget
- python-requests
- PowerShell
- Empty User-Agent strings

---

## MITRE ATT&CK

Technique:

- T1071.001 – Application Layer Protocol: Web Protocols

Tactic:

- Command and Control

---

## Investigation Steps

1. Identify the source host.
2. Review the destination host.
3. Inspect the requested URI.
4. Determine whether the tool usage is authorized.
5. Correlate with DNS, connection, and IDS logs.
