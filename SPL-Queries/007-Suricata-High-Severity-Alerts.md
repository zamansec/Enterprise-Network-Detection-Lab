# Detection 007 - High Severity Suricata Alerts

## Objective

Identify high-severity Intrusion Detection System (IDS) alerts generated by Suricata.

---

## Data Source

Suricata

Log:

- eve.json

---

## SPL Query

```spl
index=suricata sourcetype=suricata
event_type=alert
| where alert.severity<=2
| table _time src_ip dest_ip proto alert.signature alert.category alert.severity
| sort - _time
```

---

## Explanation

This query filters IDS alerts with a severity of 1 or 2 (highest severity levels in many Suricata rule sets).

The output includes:

- Alert time
- Source IP
- Destination IP
- Protocol
- Signature
- Category
- Severity

---

## MITRE ATT&CK

This detection is signature-based and may map to multiple ATT&CK techniques depending on the triggered rule.

---

## Investigation Steps

1. Review the alert signature.
2. Validate the source and destination IP addresses.
3. Correlate with Zeek connection logs.
4. Review DNS and HTTP activity.
5. Determine whether the alert is a true positive or false positive.
6. Escalate confirmed malicious activity.
